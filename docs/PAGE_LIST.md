# PAGE_LIST: Web 页面与 Bot 消息清单

## 1. 范围说明

MVP 包含 Telegram Bot 消息入口和 Web 测试页面。Bot 不承担逐题作答，Web 页面承担测试、提交和解析展示。

## 2. Telegram Bot 消息

| 名称 | 用途 | 主要内容 | 状态 |
|---|---|---|---|
| `/start` 欢迎消息 | 告知用户如何输入主题 | 日文说明、输入示例 | MVP |
| `/help` 帮助消息 | 告知用户如何创建测试 | 日文说明、主题示例、链接说明 | MVP |
| 范围解析失败消息 | 用户输入无法匹配任何题目 | 日文错误说明、2-3 个相近主题建议 | MVP |
| 测试链接消息 | 返回 Web 测试链接 | 日文说明、`/quiz/{token}` 链接 | MVP |
| 系统错误消息 | AI、DB、选题或发送失败 | 日文通用错误说明 | MVP |

## 3. Web 页面

| 页面 | 路径 | 用途 | MVP |
|---|---|---|---|
| 测试页面 | `/quiz/[token]` | 作答或查看已提交结果 | 是 |
| token 错误页 | `/quiz/[token]` 内状态 | 展示 token 无效、过期或系统错误 | 是 |
| 后续管理后台 | 未定 | 查看用户、题库、统计、日志 | 否 |

## 4. 测试页面组件

| 组件 | 用途 | 状态 |
|---|---|---|
| 顶部进度栏 | 显示当前题号、已答数、总题数、进度条 | MVP |
| 题目区域 | 渲染题干 Markdown，包含图片 | MVP |
| 选项区域 | 渲染选项 Markdown，支持图片选项 | MVP |
| 上一题/下一题按钮 | 顺序切换题目 | MVP |
| 题号导航入口 | 打开题号导航抽屉或列表 | MVP |
| 题号导航抽屉/列表 | 展示 1-20 题答题状态并支持跳转 | MVP |
| 提交按钮 | 20 题答完后启用 | MVP |
| 未答提示 | 提示未完成题号或数量 | MVP |
| 成绩摘要 | 显示正确数、错误数、正确率 | MVP |
| 测试构成摘要 | 提交后显示指定范围题、补强题构成 | MVP |
| 错题解析列表 | 默认展示错题的答案和解析 | MVP |
| 展开全部解析 | 显示全部 20 题解析 | MVP |
| 来源 URL | 提交后显示每题原题链接 | MVP |

## 5. 页面状态

| 状态 | 页面行为 |
|---|---|
| 未提交 | 可选择答案、切换题目、修改答案、提交 |
| 未答完 | 提交按钮禁用或点击后提示 |
| 提交中 | 禁用重复操作 |
| 已提交 | 锁定答案，显示结果和解析 |
| token 无效 | 显示日文错误消息 |
| token 过期 | 显示日文过期提示，要求重新从 Telegram 创建测试 |
| 加载失败 | 显示重试提示 |

## 6. 数据依赖

| 页面/组件 | 数据来源 |
|---|---|
| 测试页面初始化 | `GET /api/quiz/{token}` |
| 题干/选项/解析 | `fe_siken_questions.sqlite.question_details` |
| 图片 | `/assets/fe-siken/...` 静态资源，`question_assets` 兜底 |
| 提交答案 | `POST /api/quiz/{token}/submit` |
| 已提交结果 | `app.sqlite` 提交记录 + 题库详情 |
| 统计更新 | `app.sqlite` 用户统计表 |

## 7. UI 文案语言

- Bot 文案：日文。
- Web UI 文案：日文。
- 题目内容：题库日文原文。
- 用户输入：允许中文或日文。

## 8. 响应式与主题

- 页面移动端优先，同时适配桌面。
- 小屏题号导航使用底部 Sheet。
- 大屏题号导航使用右侧抽屉。
- 第一版只支持浅色模式。

## 9. 非 MVP 页面

- 管理后台。
- 配置管理页面。
