## 技术原则

优先使用成熟工具，不要手写已有成熟方案：

- Web 框架：Next.js + React + TypeScript
- UI：基于 Figma 设计实现，可使用 Tailwind CSS / shadcn/ui / Radix UI 中成熟组件
- 表单与校验：React Hook Form + Zod
- API 校验：Zod
- SQLite ORM / migration：Drizzle ORM + Drizzle Kit
- Telegram Bot：grammY 或 Telegraf，优先选择更适合 webhook 部署的成熟库
- OpenAI API：官方 OpenAI SDK
- Markdown / HTML 安全渲染：成熟 parser / sanitizer，不要手写解析器
- 测试：
  - 单元测试：Vitest
  - API 测试：Vitest / Supertest
  - E2E：Playwright
- Docker：
  - `web` 容器
  - `bot` 容器
  - `edge` 内部 Nginx 容器
  - 外部 VPS Nginx + Certbot 负责 HTTPS

## 任务拆分规则

请按依赖顺序拆分，不要直接进入实现代码。

每个任务必须足够小，方便后续逐步执行：

- 一个任务只解决一个清晰目标
- 每个任务必须有验证方式
- 涉及代码的任务必须先写测试或定义可验证行为
- 后端 API 与前端页面要分开任务
- 前后端对接要单独任务
- Docker 部署要单独任务
- 数据迁移和种子数据要单独任务
- Telegram webhook 验证要单独任务
- token 首次提交记录规则要单独任务

## 必须覆盖的业务规则

请确保 TodoList 覆盖以下规则：

- 用户从 Telegram Bot 进入
- Bot 根据用户输入创建 `/quiz/{token}` 链接
- token 绑定 Telegram 用户
- 用户在 Web 页面完成 20 题
- 每次出题：
  - 15 题来自用户指定范围
  - 5 题来自错题 / 低正确率主题 / 高权重主题
- 前 15 题也优先选择未做过或错得多的题
- 提交前不显示答案、解析、来源 URL
- 必须答完 20 题才能提交
- token 首次提交才写入用户历史
- 后续重复访问只读展示结果，不重复写历史
- 未提交 token 有效期 7 天
- 过期未提交 token 保留 30 天后清理
- 已提交结果长期可访问
- 移动端题号导航为 bottom sheet
- PC 端题号导航为常驻右侧栏
- 结果页必须展示：
  - 正答率
  - 正确数
  - 错误数
  - 出题构成
  - 原题
  - 全部选项
  - 用户答案
  - 正确答案
  - 解説
  - 出典 URL
- PC 结果页左侧展示全部题号及正误状态，右侧展示当前选中题目详解
- 移动端结果页为单列布局
