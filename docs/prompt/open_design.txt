你将作为产品设计师，为一个「基本情報技術者試験 科目A」在线演习系统设计 MVP 版 Web 测试体验。

## 产品背景

本产品面向准备参加日本「基本情報技術者試験」的学习者。用户通过 Telegram Bot 输入想练习的主题，系统生成一个 Web 测试链接。用户在 Web 页面中完成 20 道科目A选择题，提交后查看成绩和解析。

Telegram Bot 只负责创建测试链接，实际作答、提交、结果和解析都在 Web 页面完成。

## 设计目标

设计一个移动端优先、响应式的 Web 测试页面体验，重点是：

- 清晰展示题目和选项
- 支持图片题
- 让用户明确知道当前答题进度
- 方便在 20 道题之间切换
- 提交前保持测试感，不显示答案和解析
- 提交后高效复盘错题，并可查看全部解析

## 页面范围

### 1. 测试作答页 `/quiz/{token}`

状态：token 有效且未提交。

页面要求：

- 一题一屏展示
- 顶部固定或明显展示进度信息：
  - 当前题号
  - 已答题数 / 总题数
  - 进度条
- 展示题干 Markdown 内容
- 支持题干中的图片自适应显示
- 展示四个或多个选项
- 选项需要适合移动端点击
- 用户选择答案后应有清晰选中态
- 支持上一题 / 下一题
- 支持题号导航
- 必须答完 20 题才能提交
- 未答完时提交按钮不可用，或提示未答题号
- 页面不显示计时器
- 作答时不显示来源 URL
- 作答时不显示正确答案和解析

题号导航要求：

- 小屏：从底部滑出的 bottom sheet
- 大屏：从右侧滑出的 drawer
- 展示 1-20 题网格或列表
- 区分当前题、已答、未答
- 点击题号可跳转到对应题

### 2. 提交后结果页

状态：token 已提交。

页面要求：

- 显示成绩摘要：
  - 总题数
  - 正确数
  - 错误数
  - 正确率
- 显示本次测试构成摘要：
  - 指定范围题数量
  - 补强题数量
  - 错题池来源数量
  - 低正确率主题来源数量
  - 高权重主题来源数量
- 默认展示错题解析
- 答对题默认折叠或弱化展示
- 提供“展开全部解析”操作
- 每题结果中展示：
  - 题号
  - 题干
  - 用户答案
  - 正确答案
  - 是否正确
  - 解析
  - 来源 URL
- 支持图片题解析中的图片展示
- 已提交页面不允许重新提交
- 页面应体现答案已锁定

### 3. token 错误 / 过期页

需要设计以下状态：

- token 不存在
- 未提交 token 已过期
- 系统加载失败

文案使用日文，提示用户返回 Telegram 重新创建测试。

## 视觉方向

- 移动端优先
- 响应式适配桌面
- 第一版只做浅色模式
- 风格应安静、专注、学习工具感
- 不做营销型 landing page
- 不使用夸张插画或装饰性背景
- 重点优化可读性、答题效率和复盘清晰度
- 图片题需要清晰展示，不能被裁切
- 长题干和长解析需要良好排版
- 按钮、选项、导航控件需要有明确状态

## 关键交互状态

请为以下状态提供设计：

- 加载中
- 未选择答案
- 已选择答案
- 当前题
- 已答题
- 未答题
- 提交按钮禁用
- 提交中
- 已提交
- 答对
- 答错
- 解析展开
- 解析折叠
- 图片加载失败
