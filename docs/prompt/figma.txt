请使用 Figma 插件基于 `preview/` 文件夹中的 Open Design 线稿生成精修版高保真 UI。

## 输入范围

只读取：

- `preview/` 文件夹中的线稿图
- `docs/PRD.md`
- `docs/PAGE_LIST.md`

不要读取数据库、assets 图片目录、源码或其他技术文档。

## 设计目标

将现有线稿精修为可用于前端实现的高保真 Figma 设计稿。

必须保留线稿中的核心页面结构和信息架构：

- 作答页
- 移动端题号导航 bottom sheet
- 桌面端题号导航 drawer
- 提交后结果页
- 错题解析状态
- 展开全部解析状态
- token 过期 / 无效状态

## 重要要求

保留页面结构，但不要照搬线稿的大量线框嵌套。

请将线稿中的框线结构转化为真实产品 UI：

- 不要让页面看起来像 wireframe
- 不要大量使用嵌套描边框
- 不要把每个区域都画成卡片
- 减少不必要的边框
- 使用留白、字体层级、轻量分隔线、背景色块和状态样式来组织信息
- 卡片只用于题目、选项、结果块等真正需要承载内容的区域
- 页面整体应像成熟学习工具，而不是线稿上色版

## 风格方向

采用「现代学习工具风格」：

- 移动端优先
- 浅色模式
- 安静、专注、清晰
- 适合长时间阅读和答题
- 适合日文题干、长解析和图片题
- 不做营销风
- 不做插画风
- 不做游戏化
- 不使用夸张渐变或装饰背景

## 关键页面要求

### 作答页

- 一题一屏
- 顶部显示当前题号、已答数 / 20、进度条
- 题干区域支持长日文文本和图片
- 选项按钮适合移动端点击
- 已选择选项有明确选中态
- 提交前不显示答案、解析、来源 URL
- 未答完 20 题时，提交按钮为禁用态或显示提示

### 题号导航

移动端：

- 使用 bottom sheet
- 展示 1-20 题网格
- 区分当前题、已答、未答

桌面端：

- 使用右侧 drawer
- 保持与移动端一致的题号状态表达

### 结果页

- 显示成绩摘要
- 显示测试构成摘要
- 默认展示错题解析
- 答对题弱化或折叠
- 提供展开全部解析操作
- 提交后显示用户答案、正确答案、解析、来源 URL
- 已提交状态应体现答案锁定，不能重新提交

### 错误页

包含：

- token 不存在
- token 过期
- 加载失败

页面应清楚提示用户返回 Telegram 重新创建演習。

- 正确 / 错误 / 当前 / 未答 状态标识
