执行规则：

1. 一次只处理一个 TodoList item。
2. 开始前检查 git 状态，避免覆盖用户改动。
3. 如果任务涉及实现，先写测试，并确认测试因目标功能缺失而失败。
4. 编写最小实现。
5. 运行该任务指定验证命令。
6. 运行必要的 pnpm typecheck、pnpm lint、pnpm test。
7. 如果任务影响前端页面，运行相关 Playwright 测试。
8. 验证通过后，更新对应 docs/todolist/*.md checkbox。
9. 每完成一个 TodoList item 就提交一次代码。
10. commit 使用 Conventional Commits。
11. 提交后继续下一个 item，不要询问是否继续。
12. 只有完成全部任务或遇到 blocker 才停下。

提交规则：

- 一个 TodoList item 一个 commit。
- 如果 item 过大，先拆分子任务并更新对应 TodoList。
- 不要提交数据库文件、缓存图片、preview 线稿或无关生成物，除非任务明确要求。
- 不要使用 git reset --hard、git checkout -- 或其他会丢弃用户改动的命令。
- 不要回滚非本任务产生的改动。

暂停询问用户的条件：

1. 文档之间存在冲突，且无法通过已确认决策判断。
2. 需要真实 secret、真实域名、真实 Telegram token 或真实 OpenAI key。
3. 涉及生产数据安全或部署安全。
4. 同一 blocker 连续三次无法解决。
5. 需要删除或覆盖用户未授权的文件。

最终完成标准：

- 所有 docs/todolist/*.md 任务完成，或明确标记为非 MVP / blocked。
- pnpm typecheck 通过。
- pnpm lint 通过。
- pnpm test 通过。
- pnpm test:e2e 通过。
- Docker Compose 验证通过。
- docs/todolist/10_RELEASE_CHECKLIST.md 完成。
- 最终输出完成摘要、验证结果、提交记录和仍然存在的风险。
