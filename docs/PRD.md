# PRD: FE 科目A Telegram + Web 演習システム

## 1. 项目背景

本项目服务于准备参加日本「基本情報技術者試験」的学习者。用户通过 Telegram Bot 输入想练习的主题范围，系统从本地 SQLite 题库中选择 20 道科目A题目，并生成一个 Web 测试页面链接。用户在 Web 页面中完成作答，提交后查看结果和解析。首次提交结果会记录到用户历史中，用于后续选题。

## 2. 已确认需求

- MVP 只支持 `科目A`。
- 题库使用 `fe_siken_questions.sqlite`。
- 当前科目A题目详情已完整缓存：3400 / 3400。
- 题目图片通过 `/assets/fe-siken/...` 本地静态路径展示，不使用 base64，不 hotlink 原站图片。
- Telegram Bot 是入口，但不在 Telegram 内逐题作答。
- Bot 成功解析用户范围后，直接返回 Web 测试链接。
- 用户可用中文或日文输入练习范围。
- Bot 和 Web UI 文案统一使用日文。
- 题干、选项、解析保持题库日文原文。
- 每次测试固定 20 题。
- Web 页面采用一题一屏，顶部显示回答进度。
- 必须答完 20 题才能提交。
- 提交后默认展示错题解析，可展开全部解析。
- 每个 token 只记录首次提交。
- AI Provider 使用 OpenAI API。
- Telegram webhook 使用路径 secret 与 `X-Telegram-Bot-Api-Secret-Token` 双校验。
- 未提交 token 7 天有效；已提交结果永久可看。
- 过期未提交 session 保留 30 天后删除。
- 第一版 Bot 只支持 `/start` 和 `/help` 命令。

## 3. 目标用户

| 角色 | 描述 | 权限 |
|---|---|---|
| 学习者 | 准备参加基本情報技術者試験科目A的人 | 通过 Telegram 创建测试、打开 Web 链接作答、查看结果 |
| 系统维护者 | 部署和维护系统的人 | 维护题库、配置 YAML、环境变量、备份和服务运行 |
| AI 范围解析服务 | 系统内部能力 | 解析用户输入范围，生成相近主题建议 |

## 4. 核心用例

1. 学习者在 Telegram Bot 中输入练习主题，例如 `数据库`、`データベース`、`网络`。
2. 系统先用 YAML 主题别名和题库关键词匹配。
3. 匹配不到时调用 AI 解析中文/日文自然语言。
4. 匹配成功后，系统按规则选择 20 道科目A题。
5. Bot 直接返回 Web 测试链接。
6. 学习者打开链接，在 Web 页面完成 20 题。
7. 学习者提交答案。
8. 系统保存首次提交结果，更新用户历史统计。
9. 页面展示成绩摘要、错题解析，并允许展开全部解析。

## 5. 选题规则

### 5.1 总体结构

- 每次测试固定 20 题。
- 目标结构：
  - 15 题来自用户指定范围。
  - 5 题来自补强题池。
- 如果用户指定范围不足 15 题，则范围内有多少出多少，其余从补强题池补足到 20。
- 如果用户输入范围完全匹配不到任何题目，不生成测试，Bot 返回 2-3 个相近主题建议。
- 同一次测试内不重复出题。

### 5.2 指定范围内 15 题优先级

指定范围题优先选择：

1. 用户没做过的题。
2. 用户做错过且仍在活跃错题池中的题。
3. 用户在相关主题下正确率较低的题。
4. 尽量避开最近 7 天做过的题；该天数由 YAML 配置。

### 5.3 补强 5 题来源优先级

1. 活跃错题池。
2. 低正确率主题。
3. 高权重主题清单。

低正确率主题默认规则：

- 正确率 `< 60%`
- 且累计答题数 `>= 3`
- 阈值由 YAML 配置维护。

错题池移除规则：

- 答错后进入活跃错题池。
- 后续连续答对 2 次后，从活跃错题池移除。
- 历史明细保留。

## 7. Web 测试体验

- token 链接可直接访问，不要求登录。
- token 创建时绑定 Telegram 用户。
- 页面一次主要展示 1 道题。
- 顶部展示：
  - 当前题号
  - 已答数量
  - 总题数
  - 进度条
- 默认使用上一题 / 下一题。
- 提供题号导航抽屉或列表，可跳转任意题。
- 提交前允许修改答案。
- 未提交进度仅保存在浏览器 `localStorage`。
- 页面不显示计时器。
- 作答时不显示来源 URL。
- 提交后显示来源 URL。
- 提交后结果页展示本次测试构成摘要。
- 页面采用响应式设计，移动端优先。
- 小屏题号导航从底部 Sheet 滑出；大屏题号导航从右侧抽屉滑出。
- 第一版只做浅色模式，不做暗色模式。

## 8. token 与历史记录

- 每个 token 只记录首次提交。
- 首次提交写入该 token 绑定的 Telegram 用户历史。
- 后续再次打开 token：
  - 显示题目、首次答案、正确答案和解析。
  - 不允许重新提交。
- 重复提交请求不更新用户历史、错题池或统计表。
- 若 token 被他人获取并首次提交，结果会记到 token 创建者名下。这是 MVP 已知风险。
- 未提交 token 创建后 7 天过期。
- 过期未提交 session 保留到创建后第 30 天，再由清理任务删除。
- 已提交 token 结果页永久可访问。

## 9. MVP 范围

MVP 包含：

- Telegram Bot webhook 入口。
- Telegram webhook 路径 secret + header secret 双校验。
- 日文 Bot 文案。
- `/start` 和 `/help` 命令。
- 中文/日文主题输入解析。
- YAML 主题别名表。
- AI 兜底解析和相近主题建议。
- 科目A 20 题选题规则。
- Web 测试页面。
- 本地图片资源展示。
- 首次提交记录。
- 用户历史明细与统计表实时更新。
- Docker Compose 部署文档和备份策略。
- OpenAI API 范围解析。
- app.sqlite migration 工具管理。
- 技术细节采用推荐方案：`gpt-4.1-mini`、配置重启生效、Drizzle migration、基础 API 限流。
- UI 组件和视觉细节留到 Open Design 阶段确认。

MVP 不包含：

- 科目B。
- 用户登录系统。
- 用户自定义配置。
- 管理后台。
- 多语言 UI 切换。
- 支付、订阅、排行榜。
- AI 生成题目、改写题目或生成额外解析。
